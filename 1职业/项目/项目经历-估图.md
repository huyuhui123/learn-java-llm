
### 线上贷款平台（司机贷）

**技术栈**：SpringBoot、MyBatis-Plus、Redis、策略模式、状态机、法大大电子签API

**项目描述**：公司核心的B端金融科技产品，为物流等行业客户提供从产品准入、授信审批、融资放款到贷后管理的全流程线上化贷款服务。项目核心挑战在于构建高可维护、可扩展的架构，以快速适配多变的业务场景与风控策略。

**主要贡献**：
1. **核心业务链路设计与开发**
    - 负责“产品-授信-融资-贷后”全链路核心模块开发，深刻理解金融业务逻辑。
    - 通过与授信策略系统对接，实现基于客户多维信息的自动化额度计算；与支付模块集成，完成融资款项的安全、高效发放。

2. **电子签章系统集成与自动化**
    - 主导法大大电子签章系统从零到一的集成，独立设计并实现合同数据的动态渲染与组装逻辑。
    - 将融资合同的签署流程完全线上化、自动化，单笔合同处理时间从小时级缩短至分钟级，效率提高的同时也确保了法律效力的完整性。

3. **可扩展的架构设计**
    - 采用“策略模式”封装差异化的业务规则（如不同客户类型的额度计算逻辑），使新业务场景的接入成本大幅降低。
    - 基于“枚举状态机”清晰定义和管理融资流程，使订单状态流转高度透明，显著提升系统维护与问题排查效率。

### 批量远程尽调工具

**技术栈**：SpringBoot、MyBatis-Plus、Nginx-RTMP、SRS

**项目背景**： 为解决传统线下尽调模式效率低、成本高、时空限制强的问题，主导开发了集批量问卷与实时视频访谈于一体的线上远程尽调平台，实现对大批量客户的标准化问卷分发与关键对象的面对面视频访谈。

**主要贡献**：
- **批量数据处理能力**：独立开发了批量问卷发起模块，通过元数据绑定技术，高效完成了客户信息与问卷模板的自动化匹配与任务下发。
- **实时服务搭建能力**：基于Nginx-RTMP和SRS，从零搭建了公司内部的实时音视频服务，为视频访谈功能提供了稳定、可控的技术底座。

### AI智能尽调对话平台

**技术栈：** SpringBoot, MyBatis-Plus, Redis, SSE, jvppeteer

**项目描述：** 本项目与通用大模型应用的核心区别在于，我们构建了一个基于用户授权、融合多维度私有数据的专业化尽调分析系统。系统通过自动化流程，合法获取并整合用户的司法、财务、工商等传统大模型难以触及的授权信息，并引导大模型在此基础上进行多轮、多维度对话，最终生成具备深度数据支撑的综合判断报告，实现了从“通用问答”到“专业决策”的跨越。

**主要贡献：**


1.  **授权数据与AI对话的集成**
    *   负责将用户授权后的多维度私有数据（如司法、财务信息）自动、安全地注入到大模型对话的上下文中，为AI提供传统聊天机器人无法获取的专业数据支撑。

2.  **分布式实时对话架构实现**
    *   采用`SSE`技术实现对话的流式响应，有效改善前端用户体验。
    *   利用`Redis`的发布订阅模式，构建了分布式环境下的消息协同机制，保障了多节点服务的稳定与高可用。

3.  **复杂业务逻辑的策略化封装**
    *   运用`策略模式`来封装不同消息的消费逻辑以及报告回传知识库等差异化业务规则，提升了代码的灵活性和可维护性。

4.  **复杂PDF报告生成技术方案**
    *   负责解决复杂样式报告的输出问题，通过`jvppeteer`驱动无头浏览器，将包含图表、表格的HTML页面高保真地转换为PDF文件，确保了报告格式的专业性。

### 策略平台

**项目背景：**
核心业务规则引擎平台，旨在将频繁变动的业务决策（如风控、营销规则）从硬编码中解耦。通过可视化配置流程、策略与数据接口，实现策略的灵活编排与自动化执行，显著提升业务响应效率。

**技术栈：**
Spring Boot, MyBatis-Plus, MySQL, Redis, 策略引擎

**主要贡献：**
- **策略执行监控与运维：** 负责生产环境策略的日常监控与维护，通过日志分析和系统看板跟踪策略执行状态，确保业务规则稳定运行，并能快速定位处理因接口或数据异常导致的问题。
- **策略数据管理功能开发：** 设计并实现了策略配置的导入与导出功能，支持将完整的策略流程序列化，便于跨环境迁移、备份与版本管理，提升了配置操作的安全性与效率。
- **系统架构理解：** 通过维护与开发工作，深入理解了以“接口配置-策略规则-流程编排”为核心的三层架构，能够有效分析与排查跨层的综合性问题。

### 财税尽调工具

**项目背景：**
集财务、税务、AI报告数据供给于一体的综合尽调平台。为支持新增AI报告数据流程并提升可维护性，对原有分散、耦合的代码进行重构。

**技术栈：**
Spring Boot, MyBatis-Plus, MySQL, Redis, Apache POI, 微风企API, Arthas

**主要贡献：**
- **核心流程重构**：主导系统架构重构，运用“状态枚举+策略模式”设计，将原本分散的财务、税务流程与新增的AI数据流程统一到可扩展的执行框架中。
- **功能开发与集成**：负责财务报告（基于Apache POI生成Word）的开发与微风企税务数据接口的对接，并长期维护相关核心流程。
- **深度性能优化**：使用Arthas精准定位到财务流程中因循环内反射调用导致的性能瓶颈，通过预加载字段信息，将流程执行时间从30分钟优化至5分钟。

### AI智能报告生成系统

**项目背景：**
基于Dify平台构建的智能化报告生成系统，通过工作流整合大模型与多维度私有数据源，实现数据自动分析、报告内容智能生成与格式化输出，显著提升报告产出的效率与专业性。

**技术栈：**
Dify, 大模型API, Apache POI, Spring Boot

**主要贡献：**
- **智能化分析流程搭建**：在Dify平台设计并实现自动化工作流，将企业私有数据（财务、司法等）与大模型分析能力结合，生成专业分析结论。
- **动态报告生成系统**：开发基于模板的Word报告生成功能，支持将分析结果自动渲染至预设模板，实现：
    * 产品人员可随时调整提示词优化分析方向
    * 通过不同模板快速生成风格各异的专业报告
