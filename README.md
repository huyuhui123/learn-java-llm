## 我对工作无感，但对技术的原理感兴趣
作为后端，完成需求就是我的工作，我只要把产品下达下来的需求完成了就行，之后会进行测试，用户会开始使用，我只需要后面出问题了维护一下就行，这就是我工作的定义

我对这个流程无感，只是完成任务而已，整个流程从需求下达，到前后端开发、组装到一起，我只负责后端这一小部分，用户体验其实不关我的事，这只是我的工作

用户体验差不会扣我的钱，产品方向错了也不关我的事，做得好也不会奖励我，

我对这份工作无感，我真的想做更复杂的需求吗？一开始让我感到憧憬的是技术，是能够实现想法改变现实的手段，不是为了公司产品更好的发展而学习技术的

我并不讨厌技术，也喜欢看技术，会好奇一个功能的实现是用什么技术做到的？

但为了公司写平庸的代码耗损了我对技术的兴趣，相比之下我还更想做我喜欢的游戏

但我如何谋生，我自己有能力做一个网站，后端基础能力是有的，但我是为了什么而做事

软件改变现实的方式大多都是流程数字化，但大模型不是，它是具有学习能力的

游戏重要的不是脚本，而是美术

硬件能够改变通过屏幕、按键、信号来结合物理与化学做到更多

这样想没有一个技术能脱离其他领域，比如软件工程师在建筑器材这种偏硬件的公司能做的很少， 同理，一个硬件工程师在电商、信贷这种领域能做的事情也很少

后端，更多的负责数据传输，也就是接口，http协议、json数据格式，接口的逻辑、并发、耗时都是后端在意的东西

我会好奇自己写的接口变成硬编码时具体做了什么？也会好奇它的耗时是多少？卡点在哪里？

也就是说同样是写一个接口，我可以从工作的角度看待它的逻辑，它完成就结束了

也可以从技术的角度来看它的运行情况是怎样的，是否有我不知道的组合
